cargonetsim-cli — headless runner for CargoNetSim scenarios

USAGE
    cargonetsim-cli <subcommand> [options]

SUBCOMMANDS
    run         [--all] <scenario.yml>
                                 Execute the scenario end-to-end.
                                 `--all` is the explicit Track A1
                                 selection flag; a bare scenario path
                                 is still accepted temporarily as an
                                 alias for `--all`.

    validate    <scenario.yml>   Parse + validate; exit non-zero on error.

    preview     <scenario.yml>   Load, validate, run linker, emit JSON to stdout.

    connections [--watch]        Show per-server RabbitMQ connection status.

    --help, -h                   This help.
    --version, -v                Print version and exit.

EXIT CODES
    0   success
    1   simulation run failed at runtime
    2   scenario validation failed
    3   required simulator unavailable before simulation start
    5   at least one simulator disconnected (for connections)
   64   misuse / bad command-line arguments

ENVIRONMENT
    CARGONETSIM_CLI_RUNTIME_DIR  Override the runtime state directory.
    QT_LOGGING_RULES             Qt logging filter (respected).
